# Archived Reconciliation Notes

This file used to reconcile old phase labels with the real repository state. That was important during cleanup because older docs had started to drift from the code.

The current rule is simpler and better:

- The README explains the project.
- The capability matrix says what works and what is blocked.
- The technical debt file says what still bothers me.
- The benchmark folder gives a tiny reproducible proof path.

Old phase names are not how the project should be judged anymore. Judge it by the code, tests, Docker stack, and the limits it admits.
